About Git

Git的学习笔记

在用hexo在GitHub上搭建博客时，安装了一个软件Git。安装它是为了把本地的网页上传到GitHub上。

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

分布式，每个人的电脑里都有一个原版本，当修改好后，只要把改动的部分推送给对方，不需要推送完整的文件再让对象在里面找不同。

与集中式相比，集中式有中央服务器，所有的版本都储存在中央服务器中，每一次工作和修改保存，都需要在中央服务器上下载和上传，如果没有网络就不能工作。

版本控制系统，每一次修改保存都生成一个新本版，想保留旧版本要新建一个文档，而且时间长了也不记得修改了什么地方。版本控制系统就是用来管理这些版本，记录下每一次的修改过程。

GitHub网站为开源项目免费提供Git存储。

用hexo在GitHub上搭建博客

用hexo在GitHub的免费静态空间上初步搭建博客

这篇文章总结了互联网上许多关于“用hexo在GitHub上搭建博客”的教程，文末有参考资料。

Hexo 是什么？

Hexo是一款基于Node.js的静态博客框架，依赖少且易于安装使用，可以方便的生成静态网页托管在GitHub上，是搭建博客的首选框架。Hexo同时也是GitHub上的开源项目。它将Markdown文章通过渲染引擎，在几秒内，即可利用靓丽的主题生成静态网页，再结合Git命令进行部署。这使其成为一个快速、简洁且高效的博客框架。

用Hexo的原因
  • 不可思议的快速，只要一眨眼静态文件即生成完成
  • 支持 Markdown
  • 已移植 Octopress 插件
  • 高扩展性、自订性
  • 兼容于 Windows, Mac & Linux
  • 易用，仅需一道指令即可部署到 GitHub Pages，平时使用中仅需要hexo new hexo generate hexo server hexo deploy四个命令。
  • 轻，文件少、小，易理解，方便自定义。
  • 用户多，虽然赶不上Jekyll和Octopress，但遇到什么问题都能搜索到答案，或者找到同样使用hexo的用户进行参考和咨询。

GitHub Pages是什么？

GitHub是一个具有版本管理功能的代码仓库，每个项目都有一个主页，列出项目的源文件。

GitHub Pages本用于介绍托管在 GitHub 的项目， 不过，由于它的空间免费稳定，可以被认为是用户编写的、托管在GitHub 上的静态网页。

GitHub 提供模板，允许站内生成网页，但也允许用户自己编写网页，然后上传。有意思的是，这种上传并不是单纯的上传，而是会经过Hexo或Jekyll等程序的再处理。

我们可以用Hexo创建自己的博客，托管到github，绑定自己的域名，用markdown写文章。

0.注册Github + 创建新的个人仓库

用邮箱注册一个github账号，用来存放我们的网站。完成后，点击GitHub中的右上角的 + 新建一个项目仓库（new repository），仓库名应该为：yourname.github.io 。

这里的yourname使用你的GitHub帐号名称代替，这是固定写法，只有这样将来要部署到GitHub Page上时，才会被识别。

（文章接下来出现的yourname皆为你的GitHub账号名）

1.安装软件部分

1.1 GIT：

一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

安装它是为了把本地的网页文件上到GitHub上去，以后也可以直接使用Git Bash来敲命令。

和github区别在于，github有点类似于远程仓库，用于存放用git管理的各种项目。

安装时一路继续，安装完成后在命令行内输入git，查看是否成功。若提示 'git'是内部或外部命令，也不是可运行的程序或批处理文件。就说明失败了。

1.2 Node.js：

Hexo 是基于 Node.js 的第三方模块，所以缺少 Node.js 不可。安装Node.js时会包含环境变量及npm的安装。

安装后检测Node.js和npm是否安装成功，在命令行中输入 node -v 和 npm -v ：（显示版本号即安装成功）。

PS：添加国内镜像源：npm（Node Package Manager）是Node.js下的主流套件管理程式。它在Node.js v0.6.x版本之后，内建于Node系统。通过npm可以协助开发者安装、卸载、删除、更新Node.js套件，并且可以通过npm发布自己的插件。

1.3 Hexo：

Hexo就是我们的个人博客网站的框架，我们需要在电脑里创建一个文件夹（称之为博客根目录），用来存放Hexo的相关文件以及博客文章，文件夹名字随意，创建好后，进入文件夹中，右键进入 Git Bash Here ，或在Git Bash中 cd 到这个文件夹下。

注意，之后有关操作都需要在博客根目录下进行。

进入git 的控制台窗口后，输入 npm install -g hexo-cli 安装 Hexo

2.设置用户信息

成功安装后在桌面右键点击Git Bash Here

设置user.name和user.email配置信息， 在弹出界面中输入以下内容

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

3.与GitHub建立联系

创建SSH

ssh-keygen -t rsa -C "你的GitHub注册邮箱"

然后直接三个回车即可，即不设置密码。

然后在C:\Users\用户名\.ssh路径下找到名为id_rsa.pub的文件，就是你刚刚创建的密钥，右键用记事本打开然后复制全部内容。

然后进入GitHub添加密钥：右上角齿轮setting—左边列表SSH keys—将复制的内容粘贴至Key中，title随便填—Add SSH key。

在Git Bash中输入 ssh -T git@github.com 来检测GitHub公钥设置是否成功。

当出现 Hi,yourname! You've successfully......则已经成功。这时Github会给你的邮箱发送一封邮件，打开邮件确认下就好了。

PS：这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了 省去每次输入密码的步骤 ，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

4.Hexo初始化配置

在安装Hexo时创建的文件夹下右键–Git Bash Here。执行以下命令hexo init

安装依赖包：npm install

该命令会将hexo所需文件自动下载到文件夹下。

  1. node_modules是node.js 各种库的目录，
  2. public 是生成的网页文件目录（一开始可能没有）
  3. scaffolds 里面就三个文件，存储着新文章和新页面的初始设置
  4. source 是我们最常用到的一个目录，里面存放着文章、各类页面、图像等文件
  5. themes 存放着主题文件
  6. _config.yml 站点配置文件

我们需要将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上，打开站点配置文件 _config.yml（博客根目录中），划到最后，将deploy中的内容修改成下面的样子（yourname改成你GitHub的用户名）：

然后我们还需要安装部署所需插件 deploy-git

npm install hexo-deployer-git --save

然后按顺序输入以下命令

hexo clean
hexo generate —- hexo g 缩写
hexo deploy —-hexo d

全部完成后访问，恭喜，你的博客已经搭建完成，博客地址：yourname.github.io（yourname改成你GitHub的用户名）

6.写文章，发布文章

首先在博客根目录下右键打开Git Bash，安装一个扩展 npm i hexo-deployer-git。
然后输入hexo new post "article title" ，新建一篇文章。

编写完markdown文件后，根目录下输入 hexo g 生成静态网页，然后输入hexo s可以本地预览效果，最后输入 hexo d 上传到github上。这时打开你的github.io主页就能看到发布的文章了。
